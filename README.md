# 🛠️ TecnoRepair Perú S.A.C. - Sistema de Gestión de Reparaciones

Aplicación de escritorio desarrollada en **Java** para gestionar el proceso de reparación de equipos tecnológicos, permitiendo administrar clientes, equipos, órdenes de servicio, diagnósticos, presupuestos, pagos e historial de reparaciones.

> **Proyecto desarrollado con fines académicos para el curso de Taller de Programación.**

---

## 📖 Descripción

Este sistema fue desarrollado como **Proyecto Final del curso Taller de Programación** de la carrera de Ingeniería de Sistemas e Informática.

El objetivo del proyecto fue aplicar los conocimientos adquiridos durante el curso, especialmente los relacionados con:

- Programación Orientada a Objetos (POO).
- Manejo de archivos CSV.
- Modularización del código.
- Diseño de aplicaciones de consola en Java.
- Gestión de información mediante estructuras de datos.
- Buenas prácticas de programación.

La aplicación simula el funcionamiento de un sistema utilizado por una empresa dedicada a la reparación de equipos tecnológicos.

---

## ⚠️ Aviso

Este proyecto fue desarrollado exclusivamente con fines académicos como **Proyecto Final del curso Taller de Programación** de la carrera de Ingeniería de Sistemas e Informática.

La empresa presentada en este proyecto, **TecnoRepair Perú S.A.C.**, fue creada como un caso de estudio para simular el funcionamiento de un sistema de gestión de reparaciones.

Aunque pueden existir empresas reales con nombres similares, este proyecto no tiene ninguna relación, afiliación ni representación de ninguna empresa existente. Todos los datos utilizados (clientes, técnicos, equipos, direcciones, órdenes de servicio y demás registros) fueron creados únicamente con fines de aprendizaje y demostración.

---

## ✨ Funcionalidades

- Inicio de sesión de usuarios.
- Registro de nuevos usuarios.
- Gestión de clientes.
- Gestión de equipos.
- Creación de órdenes de servicio.
- Actualización del estado de las reparaciones.
- Registro de diagnósticos.
- Cálculo automático del presupuesto.
- Registro de pagos parciales o totales.
- Generación de comprobantes.
- Historial por cliente.
- Historial por equipo.
- Reporte de órdenes pendientes.
- Reporte de ingresos acumulados.
- Persistencia de datos mediante archivos CSV.

---

## 🛠️ Tecnologías utilizadas

- Java
- Programación Orientada a Objetos (POO)
- NetBeans IDE
- Archivos CSV para persistencia de datos
